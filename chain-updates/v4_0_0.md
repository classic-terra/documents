## Key Changes
Release: https://github.com/classic-terra/core/releases/tag/v4.0.0

### ⚠️ SDK v0.53 notice: transaction logs deprecated (compatibility update)
With the upgrade to **Cosmos SDK v0.53** (ABCI 2.0 / CometBFT), the SDK no longer constructs **`sdk.TxLogs`** by default. This is an **upstream change** introduced with ABCI 2.0.

#### What changed upstream
- `logs` and `raw_log` are **deprecated** and are **no longer constructed by default**.
- The canonical output for tx results is now **events** (protocol-level ABCI events).

References:
- Cosmos SDK PR: https://github.com/cosmos/cosmos-sdk/pull/15845
- CosmJS changelog: https://github.com/cosmos/cosmjs/blob/main/CHANGELOG.md#deprecated-4

#### What did not change
- Contracts still emit events.
- Events are still indexed and queryable.
- Explorers/indexers that already consume events are not affected.

#### ✅ Terra Classic resolution
To reduce ecosystem breakage while aligning with upstream behavior:
- Terra Classic adds a **compatibility layer on transaction queries** (`getTx`) to reconstruct **legacy `logs` from events**.
- **Broadcast responses remain upstream** (no `logs`).

Implementation:
- TxLogsMiddleware PR: https://github.com/classic-terra/core/pull/635

#### What this means / required actions (apps & integrators)
- ✅ Apps that rely on **tx queries** should continue to work, but are strongly encouraged to migrate to **events** as canonical.
- ⚠️ Apps that rely on `logs` from **broadcast** responses will be affected. Migration path:
  1) broadcast tx
  2) wait for inclusion
  3) query by hash
  4) read `events` (or reconstructed `logs` via query)

New expected usage:
- Instead of `tx.logs[0].events`, use `tx.events`.

(Original note reference: https://hackmd.io/@orbitlabs/rJRuMsfSWl)
